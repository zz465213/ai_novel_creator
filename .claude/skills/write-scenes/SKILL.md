# /write-scenes — 場景撰寫與進度管理

## 指令目標

根據三層上下文（寫作標準 → 小說配置 → 任務大綱）精確執行逐場戲的撰寫任務，將產出的章節正文存放於正確的小說目錄中，並自動更新進度。

---

## 執行前：確認目標小說與任務

1. 若有多部小說存在，列出 `.novel-os/novels/` 下所有目錄，請使用者選擇
2. 確認工作路徑：`.novel-os/novels/[編號-故事名稱]/`
3. 章節輸出路徑：`.novel-os/novels/[編號-故事名稱]/VolN/chN.md`
   - `N`（卷號）依當前任務的所屬卷決定
   - `N`（章號）為全書連續編號，不因換卷重置

---

## 上下文載入清單

**每次撰寫場景前，依序讀取以下檔案**（若存在）：

| 優先級 | 檔案路徑 | 用途 |
|--------|---------|------|
| 高 | `[小說目錄]/premise-lite.md` | 故事核心快速參照 |

## 執行步驟

### 步驟一：確認當前任務

1. 讀取 `tasks.md`，找到第一個狀態為 `[ ]`（未開始）的任務
2. 若當前任務為某卷**第一章**，先讀取前一卷的 `vol-summary.md`（若存在）作為跨卷銜接參照
3. 向使用者確認：
   ```
   準備撰寫：任務 [編號] — [場景名稱]
   輸出至：VolN/chN.md
   所屬卷：第N卷
   POV：[角色名]
   場景目標：[目標描述]
   字數規格：最少 XXXX 字，目標 XXXX 字

   確認開始撰寫？或是指定其他任務？
   ```
4. 若使用者指定特定任務或章節編號，直接跳到該任務

---

### 步驟二：場景撰寫規則

#### 字數強制執行機制

1. 撰寫前從 `tasks.md` 讀取該話的「字數規格」欄位（最少/目標/上限）
2. 完成初稿後自行估算字數
3. **若低於最少字數要求，必須繼續擴寫，不得輸出**
4. 擴寫方向優先順序：
   a. 深化主要衝突的情感層次
   b. 補充感官細節與環境描寫
   c. 加入角色內心獨白
   d. 強化對話的潛台詞與肢體反應
5. 輸出時在撰寫備忘欄標注**實際估算字數**

#### 風格遵循原則

- **視角鎖定**：嚴格依照 `writing-style.md` 中的 POV 設定，不在場景中途切換視角
- **時態一致**：全場景維持統一時態
- **聲音匹配**：文字的語氣、節奏與用詞應符合偵測到的作者聲音
- **對話原生感**：每個角色的說話方式應符合其 `character-profiles.md` 中的設定

#### 場景結構要求

每個場景應包含：

1. **進場**：迅速建立場景的地點、時間與人物（避免冗長描述）
2. **核心衝突/目標**：每場戲必須有清晰的場景層級目標
3. **進展或轉折**：場景結束時，角色的狀態或認知應有所改變
4. **結尾鉤子**（非最終章）：以問題、揭示或情感張力結束，引導讀者繼續

#### 角色一致性自查

撰寫對話與行為時確認：
- [ ] 這個角色在此情況下真的會這樣說話/行動嗎？
- [ ] 角色的反應是否符合其核心傷口與恐懼？
- [ ] 對話是否推進情節或揭示角色？（避免「廢話對話」）
- [ ] 角色是否使用了其禁用詞或不符合個性的表達？

---

### 步驟三：輸出格式

**章節正文**寫入：
```
.novel-os/novels/[編號-故事名稱]/VolN/chN.md
```

> 卷資料夾若不存在，先建立後再寫入。

章節檔案格式：
```markdown
# 第[X]章：[章節標題（若有）]

[場景正文]

---
<!-- 草稿狀態：[初稿/修稿中/定稿] -->
<!-- 所屬卷：第N卷 -->
<!-- 字數：約 XXXX 字 -->
```

場景完成後，在**對話中**（非寫入檔案）附上撰寫備忘：
```
---
[撰寫備忘]
- 本場景達成：[場景目標完成度]
- 角色弧線進展：[主要角色的狀態變化]
- 下一場戲銜接建議：[開場方向]
- 需要作者確認：[任何有疑問的創意選擇]
```

---

### 步驟四：自動更新進度

場景獲得使用者確認後：

1. 更新 `tasks.md`：將該任務標記為 `[x] 已完成`，記錄完成日期
2. 更新 `writing-plan.md`：標記對應里程碑進度
3. 在 `decisions.md` 中記錄撰寫過程中的重要創意決策（若有）
4. 偵測是否為當前卷的最後一章 → 若是，執行步驟五
5. 若非末章，告知下一個待完成任務（任務編號 + 場景名稱）

---

### 步驟五：卷完成處理（當偵測到當前卷最後一章完成時觸發）

#### 5-1. 生成卷總結

自動建立 `VolN/vol-summary.md`，格式如下：

```markdown
# [小說標題] 第N卷總結

## 卷基本資訊
- 卷名：[卷名]
- 章節範圍：chX – chY
- 總字數：約 XX 萬字
- 完成日期：[日期]

## 情節摘要（500 字以內）
[本卷主要事件的壓縮摘要，涵蓋起承轉合]

## 角色狀態（卷末）
| 角色 | 卷末狀態/位置 | 本卷重要變化 |
|------|------------|------------|

## 關鍵伏筆紀錄
- 本卷設置待回收的伏筆：
- 本卷回收的前卷伏筆：

## 下一卷銜接點
[卷末懸念描述 + 建議的下一卷開場方向]
```

> 用途：開始撰寫下一卷時載入此摘要，**取代**前卷所有章節全文，大幅縮短上下文。

#### 5-2. 上下文刷新建議

生成卷總結後，向使用者顯示：

```
✅ 第N卷完成！共 X 章，約 XX 萬字。
已生成卷總結：VolN/vol-summary.md

─────────────────────────────────
💡 建議：開始第N+1卷前，考慮刷新上下文
─────────────────────────────────
原因：目前對話已累積大量前卷章節內容，
      繼續會話可能導致 AI 記憶壓縮、字數縮減。

選項：
  [A] 開新對話（推薦）
      → 新對話開始時提供：
         1. premise-lite.md
         2. VolN/vol-summary.md（代替前卷章節）
         3. tasks.md（第N+1卷的任務）

  [B] 繼續目前對話
      → 我將自動載入 vol-summary.md 並清除前卷章節記憶
      → 注意：長時間累積後仍可能出現品質衰減

請選擇 A 或 B，或直接說「繼續寫」預設選 B。
```

## 常見使用情境

| 情境 | 使用者說法 | AI 行為 |
|------|-----------|---------|
| 按序撰寫 | 「繼續寫」 | 讀取 tasks.md → 找下一個未完成任務 → 確認 → 撰寫 |
| 指定場景 | 「寫任務 005」 | 定位任務 005 → 讀取相關設定 → 撰寫 |
| 指定章節 | 「寫第三章」 | 找對應 Vol1/ch3.md 的任務 → 撰寫 |
| 接續前章 | 「接著第三章結尾繼續」 | 讀取 Vol1/ch3.md 最後兩段 → 確認語氣 → 撰寫 |
| 跨卷開始 | 「開始第二卷」 | 讀取 Vol1/vol-summary.md → 確認銜接點 → 撰寫 Vol2/ch首章 |
| 重寫場景 | 「重寫任務 003，讓主角更猶豫」 | 讀取原場景 → 調整情感基調 → 重寫 |

## 品質把關

**當寫作卡住時：**
- 情節卡住 → 回頭檢查 `story-outline.md` 是否有漏洞
- 角色感覺不對 → 重讀 `character-profiles.md` 的核心傷口與慾望
- 風格感覺偏移 → 重讀 `writing-style.md` 並對照前幾章的代表性段落

**若發現任務描述與故事邏輯有衝突：先暫停並告知使用者，不要強行寫下去。**

---

## 會話管理

若一次會話無法完成整個場景：
1. 在停筆處加入標記：`<!-- 停筆點：[簡短說明當前狀態] -->`
2. 將已完成部分儲存為 `chapter-[編號]-draft.md`
3. 告知使用者：「已完成至[描述]，下次說『接著上次的場景』即可繼續」

#### 跨對話字數一致性（長篇防衰減）

若當前話數為**第 8 話之後**，撰寫前**必須**執行以下步驟：

1. 重讀 `tasks.md` 的字數規格欄位，重新錨定本話目標
2. 讀取前一話章節的最後兩段，校準語氣與節奏
3. 確認字數規格後再下筆

> 此機制防止長篇累積後因對話記憶壓縮導致字數縮減的問題。字數規格以 `tasks.md` 為唯一權威，對話記憶不可覆蓋此設定。
