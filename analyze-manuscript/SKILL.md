# /analyze-manuscript — 既有稿件導入與分析

## 指令目標

讓 AI 深度理解作者已完成的內容，將其轉化為 Novel-OS 的結構，建立自包含的小說專案目錄，讓作者無需從零開始配置即可繼續創作。

### 目錄命名規則
- 格式：`[三位數編號]-[kebab-case英文名]`
- 範例：`001-the-last-bookshop`
- 編號：檢查 `.novel-os/novels/` 下現有目錄，取最大編號 +1

---

## 執行前：確認輸入來源

詢問使用者提供以下任一種輸入（或組合）：

1. **直接貼上文字**：章節或片段（建議至少 500 字以獲得準確分析）
2. **指定檔案路徑**：現有章節的本地路徑
3. **描述現況**：簡述寫了多少、寫到哪裡

---

## 執行步驟

### 步驟一：掃描分析（四個面向）

#### 1. 檔案結構分析
- 檢查現有目錄與章節組織方式
- 識別命名規則與章節劃分邏輯
- 統計已完成字數與章節數

#### 2. 寫作風格偵測
分析提供的文字樣本，識別：

| 面向 | 偵測內容 |
|------|---------|
| 敘事視角（POV） | 第一人稱 / 第三限知 / 第三全知 |
| 時態 | 過去式 / 現在式 |
| 敘事聲音 | 語氣（嚴肅/幽默/詩意）、句子長度傾向 |
| 類型特徵 | 推理 / 奇幻 / 文學 / 愛情等 |
| 對話風格 | 比例、標記方式、角色語言個性 |

#### 3. 既有文件盤點
- 讀取現有的角色筆記、設定集、大綱（若有）
- 識別已建立的角色、地點、重要情節
- 標記需要補充或更新的資訊

#### 4. 進度整合
- 將已完成章節對應到故事結構（起 / 承 / 轉 / 合）
- 估算剩餘工作量
- 識別目前停筆的位置與原因（若能判斷）

---

### 步驟二：產出分析報告

在建立檔案前，先向使用者呈現分析摘要：

```
## 稿件分析報告

### 偵測到的寫作風格
- 視角：[結果]
- 時態：[結果]
- 類型：[結果]
- 聲音特徵：[描述]

### 進度概況
- 已完成：[字數/章節數]
- 故事推進到：[位置描述]
- 估計完成度：[百分比]

### 識別到的角色
[清單]

### 識別到的設定/世界觀元素
[清單]

### 建議補充的資訊
[哪些配置需要使用者手動確認]
```

詢問使用者確認後再繼續。

---

### 步驟三：建立 Novel-OS 結構

確認無誤後，建立以下完整目錄結構：

```
.novel-os/novels/[編號-故事名稱]/
├── premise.md             ← 根據分析推導，未知欄位標記 [待補充]
├── premise-lite.md        ← 精簡版摘要，300 字以內
├── writing-style.md       ← 偵測到的風格 + 原文代表性例句
├── writing-plan.md        ← 已完成章節標記 [x]，後續待規劃
├── decisions.md           ← 記錄分析時發現的重要創意選擇
├── story-outline.md       ← 根據現有內容推導，未知部分標記 [待規劃]
├── tasks.md               ← 已完成場景標記 [x]，待寫場景標記 [ ]
├── sub-specs/
│   ├── character-profiles.md   ← 從稿件識別的角色資訊
│   └── world-building.md       ← 從稿件識別的設定元素
└── chapters/                   ← 遷移或連結現有章節
    ├── chapter-01.md
    └── ...
```

**chapters/ 處理原則**：
- 若原始章節已有規範命名（如 `chapter-01.md`）：直接複製或建立符號連結
- 若命名不規範：詢問使用者是否要重新命名，或保持原檔名並在 `tasks.md` 中標記對應關係

---

### 步驟四：給出行動建議

完成後提供：

1. **需要使用者補充的清單**：哪些 `[待補充]` 欄位最重要
2. **下一步建議**：
   - 若大綱不完整 → 執行 `/create-outline` 補充規劃
   - 若大綱已夠清楚 → 直接執行 `/write-scenes` 繼續撰寫
3. **風格校對提示**：若偵測到的風格有不一致之處，指出具體位置

---

## 注意事項

- **不覆蓋原有內容**：所有分析結果寫入新建的 `.novel-os/novels/[編號-故事名稱]/` 目錄，不修改原始章節檔案
- **標記不確定性**：無法確定的資訊一律標記 `[待確認]` 而非猜測
- **尊重作者選擇**：若偵測到「非常規」的寫作手法（如刻意的句子片段），視為風格特色而非錯誤
