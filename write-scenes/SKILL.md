# /write-scenes — 場景撰寫與進度管理

## 指令目標

根據三層上下文（寫作標準 → 小說配置 → 任務大綱）精確執行逐場戲的撰寫任務，將產出的章節正文存放於正確的小說目錄中，並自動更新進度。

---

## 執行前：確認目標小說與任務

1. 若有多部小說存在，列出 `.novel-os/novels/` 下所有目錄，請使用者選擇
2. 確認工作路徑：`.novel-os/novels/[編號-故事名稱]/`
3. 章節輸出路徑：`.novel-os/novels/[編號-故事名稱]/chapters/`

---

## 執行步驟

### 步驟一：確認當前任務

1. 讀取 `tasks.md`，找到第一個狀態為 `[ ]`（未開始）的任務
2. 向使用者確認：
   ```
   準備撰寫：任務 [編號] — [場景名稱]
   輸出至：chapters/chapter-[編號].md
   POV：[角色名]
   場景目標：[目標描述]
   預估字數：約 XXXX 字
   
   確認開始撰寫？或是指定其他任務？
   ```
3. 若使用者指定特定任務或章節編號，直接跳到該任務

---

### 步驟二：場景撰寫規則

#### 風格遵循原則

- **視角鎖定**：嚴格依照 `writing-style.md` 中的 POV 設定，不在場景中途切換視角
- **時態一致**：全場景維持統一時態
- **聲音匹配**：文字的語氣、節奏與用詞應符合偵測到的作者聲音
- **對話原生感**：每個角色的說話方式應符合其 `character-profiles.md` 中的設定

#### 場景結構要求

每個場景應包含：

1. **進場**：迅速建立場景的地點、時間與人物（避免冗長描述）
2. **核心衝突/目標**：每場戲必須有清晰的場景層級目標
3. **進展或轉折**：場景結束時，角色的狀態或認知應有所改變
4. **結尾鉤子**（非最終章）：以問題、揭示或情感張力結束，引導讀者繼續

#### 角色一致性自查

撰寫對話與行為時確認：
- [ ] 這個角色在此情況下真的會這樣說話/行動嗎？
- [ ] 角色的反應是否符合其核心傷口與恐懼？
- [ ] 對話是否推進情節或揭示角色？（避免「廢話對話」）
- [ ] 角色是否使用了其禁用詞或不符合個性的表達？

---

### 步驟三：輸出格式

**章節正文**寫入：
```
.novel-os/novels/[編號-故事名稱]/chapters/chapter-[編號].md
```

章節檔案格式：
```markdown
# 第[X]章：[章節標題（若有）]

[場景正文]

---
<!-- 草稿狀態：[初稿/修稿中/定稿] -->
<!-- 字數：約 XXXX 字 -->
```

場景完成後，在**對話中**（非寫入檔案）附上撰寫備忘：
```
---
[撰寫備忘]
- 本場景達成：[場景目標完成度]
- 角色弧線進展：[主要角色的狀態變化]
- 下一場戲銜接建議：[開場方向]
- 需要作者確認：[任何有疑問的創意選擇]
```

---

### 步驟四：自動更新進度

場景獲得使用者確認後：

1. 更新 `tasks.md`：將該任務標記為 `[x] 已完成`，記錄完成日期
2. 更新 `writing-plan.md`：標記對應里程碑進度
3. 在 `decisions.md` 中記錄撰寫過程中的重要創意決策（若有）
4. 告知下一個待完成任務（任務編號 + 場景名稱）

| 情境 | 使用者說法 | AI 行為 |
|------|-----------|---------|
| 按序撰寫 | 「繼續寫」 | 讀取 tasks.md → 找下一個未完成任務 → 確認 → 撰寫 |

| 接續前章 | 「接著第三章結尾繼續」 | 讀取 chapter-03.md 最後段落 → 確認語氣 → 撰寫 |
| 重寫場景 | 「重寫任務 003，讓主角更猶豫」 | 讀取原場景 → 調整情感基調 → 重寫 |

## 品質把關

**當寫作卡住時：**
- 情節卡住 → 回頭檢查 `story-outline.md` 是否有漏洞
- 角色感覺不對 → 重讀 `character-profiles.md` 的核心傷口與慾望
- 風格感覺偏移 → 重讀 `writing-style.md` 並對照前幾章的代表性段落

**若發現任務描述與故事邏輯有衝突：先暫停並告知使用者，不要強行寫下去。**

---

## 會話管理

若一次會話無法完成整個場景：
1. 在停筆處加入標記：`<!-- 停筆點：[簡短說明當前狀態] -->`
2. 將已完成部分儲存為 `chapter-[編號]-draft.md`
3. 告知使用者：「已完成至[描述]，下次說『接著上次的場景』即可繼續」
